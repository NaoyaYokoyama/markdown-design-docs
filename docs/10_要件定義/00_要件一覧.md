# 10_要件定義/00_要件一覧.md

<details>
<summary>入力ルール（優先度・状態・ID）</summary>

**優先度**

- 急：障害対応など、通常業務より優先して対応する
- 高：通常業務内の最優先案件
- 中：優先的に対応する
- 低：時間があれば対応する

**状態**

- 未着手：作業開始前
- 要件定義：要件定義中
- 設計：設計作業中
- 製造：開発作業中
- UT：単体テスト
- IT：結合テスト
- ST：システムテスト
- UAT：ユーザー受入テスト
- リリース待：開発・テスト完了、リリース待ち
- 完了：リリース済み（リリース日も入力）
- 保留：一時停止

**その他**

- IDは「RQ001-01」の形式で管理します。
- 「RQ001」は機能単位、「-01」は機能内の要件を表します。
- 要求元には、要望を提出した部署、組織、個人を記載します。
- 担当者には、主担当者を記載します。
- リリース日は、リリース前は予定日、リリース後は実績日を入力します。
- 備考には極力入力しないこと（一覧の可読性を保つため）

</details>
