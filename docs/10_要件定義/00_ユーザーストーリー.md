# 10_要件定義/00_ユーザーストーリー.md

<details>
<summary>入力ルール（優先度・状態・ID）</summary>

**優先度**

- 高：最優先案件
- 中：優先的に対応する
- 低：時間があれば対応する

**状態**

- 未着手：作業開始前
- 要件定義：要件定義中
- 設計：設計作業中
- 製造：開発作業中
- UT：単体テスト
- ST：結合テスト
- UAT：ユーザー受入テスト
- リリース待：開発・テスト完了、リリース待ち
- 完了：リリース済み（リリース日も入力）
- 保留：一時停止

**その他**

- IDは「US0001-01」の形式で管理します。
- 「US0001」は機能単位、「-01」は機能内のユーザーストーリーを表します。
- 要求元には、要望を提出した部署、組織、個人を記載します。
- 担当者には、ユーザーストーリーの主担当者を記載します。
- リリース日は、リリース前は予定日、リリース後は実績日を入力します。

</details>
